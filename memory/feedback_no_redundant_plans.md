Do NOT write a formal plan document when the spec is already approved and the user says "implement". Go straight to writing code.

**Why:** User gets frustrated when tokens are wasted on intermediate artifacts (plan docs) after the design is already agreed upon.

**How to apply:** After brainstorming + spec approval, invoke executing-plans or just start coding directly. Skip the writing-plans step if the user says "implement", "wdroz", "napisz kod", or equivalent.
